# M3 — Verify 阶段计划

> 状态：草拟 2026-05-27（DCP-2 有条件通过后）。M3 目标见下；按依赖排序的增量 M3-1…M3-7。
> 承接 DCP-2 四项附条件（见 `roadmap.md` DCP 日志）+ M3 本职（eval 基线 / 回归 / 试用反馈）。

## 目标
把系统从"能端到端跑通"（M2 已证）推进到"**在真实多源数据下达上线门槛、可回归监控、经试用反馈**"，闭合 DCP-2 附条件，准出 **DCP-3 → M4 Launch**。

## DCP-3 准入（M3 退出标准，预设）
- 上线门槛指标在**多源（非纯 arXiv）评测集**上达标：引用可达性 100%、幻觉率 ≤2%、校验器准召达 `eval-criteria` 线；
- 成本阈值**实测定稿**（按中转站真实账单），单份报告在阈内；
- 回归测试 + CI（typecheck/test/build/eval 抽样）绿，回归告警接线；
- 稳定性：全量分析不超时、长任务有重试 + 失败告警；
- 试用反馈闭环完成 ≥1 轮，无 🔴 未决。

## 增量（按依赖排序）

### M3-1 · 稳定性地基（最先，解除后续阻塞）— ✅ 核心完成 2026-05-27
- **F4 analyzer 大池分批** ✅：`chunkByChars` 按正文字符预算（默认 30k）切批，逐批分析合并；单批小池行为不变。
- **超时修复** ✅：45s→120s（env `LLM_TIMEOUT_MS`）——live 确认揪出"45s 对 Opus 8k 输出太短、重试也只 45s → 合法慢生成永不成功"的第二根因。
- **live 确认** ✅：富正文 12 条 / 26.5 万字 → 9 批，120s 下 290s 完成、29 洞察（之前必超时）。
- 重试（`maxRetries 2`）+ 失败捕获（`runJob` → `Run.error` + pino）已在位；**告警 channel** 待运维基础设施（M4/运维）。

### M3-2 · 评测集升级（DCP-2 附条件②）— ✅ 机制+重测完成 2026-05-27（**揭示退化**）
- **多源评测数据** ✅：方案 B（本地快照不入仓）——`eval:build-local` 从 .data 抽多源富内容 + `A1_QUALITY_FILE` 接入；只提交配方/指标。
- **基线固化** ✅：`evals/baseline.json`（arXiv 基线 + 多源段）。
- **标注人工校验** ✅：A1 复验已固化 20/20（`a1-label-audit-2026-05-26.md`）。
- **多源重测结果（重要）**：引用可达性 **98.3%→24.1%**（analyzer 在长/口语化内容上拼接/改写 quote）；security 原始内容触发**安全拒答**。→ **A1 可溯源在脏多源内容下不达标**，修复转 M3-6。详见 `a1-runs.md`。

### M3-3 · 成本 / 性能标定（DCP-2 附条件①）
- **成本实测**：受控样本跑（M3-1 稳定后），对**中转站真实账单**核对，验证 / 收紧重标阈值（brief¥5 / deep_dive¥15 / initial_digest¥30）；定 thinking on/off 成本-安全取舍。
- **P50 时延**：端到端（采集→报告）实测对照 charter「P50 ≤10 分钟」。
- **已知输入（M3-1 live 实测）**：富正文 12 条 / 9 批 analyze ≈ **¥6.26**（vs 6 条 arXiv 单批 ¥0.53）——**分批使 SYSTEM 提示重复 N 次、成本随批数上升**；prompt caching 可缓解但中转站不命中（待直连 key）。标定须区分内容形态（arXiv 摘要 vs 富正文长文）；可调 `ANALYZE_BATCH_CHARS` 在"批数/超时风险"间权衡降本。
- 交付：成本实测报告 + 阈值定稿 + 时延实测；不依赖 key（Sonnet 干净测算待将来）。

### M3-4 · 回归与 CI eval — ✅ 回归门完成 2026-05-27
- **回归门** ✅：`run-a1` 跑完对照 `baseline.json`（arXiv 基线），任一指标较基线 **降 >3pp 即 ⚠️ + 非零退出**（按 op 方向：`>=` 指标降 / `<=` 指标升）；全量非冒烟下阻断，离线验证逻辑正确。
- **CI 现实**：真模型 eval 需 key + 预算 + 中转站，**公共 CI 跑不了** → 确定性 eval 逻辑由 vitest 覆盖（无 key，CI 已跑）；真模型 eval + 回归门在**带 key 环境**（本地/手动/将来定时 job + secrets）运行。
- 交付：回归门 ✅；带 key 的定时 eval job（接 secrets）留将来。

### M3-5 · 试用反馈闭环（dogfood）
- cron 常态运行一段、团队 dogfood、收集反馈迭代；多人交叉评审固化（承接 A1 条件④单人→多人）。
- 交付：运行记录 + 反馈清单 + 迭代。依赖 M3-1/2/3（稳定 + 达标）。

### M3-6 · 质量红线收紧（A1 遗留③ + M3-2 多源退化）— 🟢 引用/拒答已显著改善 2026-05-27
- **引用纪律** ✅：`repairQuote`（对齐回连续 verbatim 子串）+ SYSTEM 规则 3 强化"逐字、宁短勿拼、≤30 字精确片段" → 多源可达性 **24%→54%→89%**。
- **安全拒答缓解** ✅：SYSTEM 框定明确"Krebs/Darknet 等是已公开安全新闻/播客，中性摘录、非操作指南" → security 主题从拒答跳过到产出 4 洞察。
- **余项**：89% < 100% 硬门槛（脏内容固有真改写，被闸门正确挡下；收益递减，留 body 清洗后续）；幻觉率随规模收紧至 ≤2%（发布门）；analyzer "数值须被引用覆盖"（#14 类）；截断守卫治本（streaming/自由文本，A1 遗留）。
