# M3 — Verify 阶段计划

> 状态：草拟 2026-05-27（DCP-2 有条件通过后）。M3 目标见下；按依赖排序的增量 M3-1…M3-7。
> 承接 DCP-2 四项附条件（见 `roadmap.md` DCP 日志）+ M3 本职（eval 基线 / 回归 / 试用反馈）。

## 目标
把系统从"能端到端跑通"（M2 已证）推进到"**在真实多源数据下达上线门槛、可回归监控、经试用反馈**"，闭合 DCP-2 附条件，准出 **DCP-3 → M4 Launch**。

## DCP-3 准入（M3 退出标准，预设）
- 上线门槛指标在**多源（非纯 arXiv）评测集**上达标：引用可达性 100%、幻觉率 ≤2%、校验器准召达 `eval-criteria` 线；
- 成本阈值**实测定稿**（按中转站真实账单），单份报告在阈内；
- 回归测试 + CI（typecheck/test/build/eval 抽样）绿，回归告警接线；
- 稳定性：全量分析不超时、长任务有重试 + 失败告警；
- 试用反馈闭环完成 ≥1 轮，无 🔴 未决。

## 增量（按依赖排序）

### M3-1 · 稳定性地基（最先，解除后续阻塞）
- **F4 analyzer 大池分批**：宽主题 / 富正文一次性灌 analyze 会超时（populate + 成本测量均撞到）→ 按 token 预算分批分析 + 合并。**这是全量 brief/deep_dive 可靠的前提。**
- **中转站韧性**：analyze 长请求超时调优 + 指数退避重试 + 连续失败告警（DCP-2 附条件③，测量期反复超时）。
- 交付：分批分析 + 重试/超时策略 + 失败告警；不依赖 key。

### M3-2 · 评测集升级（DCP-2 附条件②）
- **补非 arXiv 多源评测数据**：news/blog/播客等真实内容构建 QualityCase（依赖 M3-1 稳定采集）。
- **一致性标注集人工校验**：现标注为 AI 构造，人工校验去偏差（A1 复验已抽检 20/20 良好，扩到全量或抽样固化）。
- **基线固化**：当前指标存基线快照，供回归对比。
- 交付：多源评测集 + 人工校验标注集 + 基线；不依赖 key。

### M3-3 · 成本 / 性能标定（DCP-2 附条件①）
- **成本实测**：受控样本跑（M3-1 稳定后），对**中转站真实账单**核对，验证 / 收紧重标阈值（brief¥5 / deep_dive¥15 / initial_digest¥30）；定 thinking on/off 成本-安全取舍。
- **P50 时延**：端到端（采集→报告）实测对照 charter「P50 ≤10 分钟」。
- 交付：成本实测报告 + 阈值定稿 + 时延实测；不依赖 key（Sonnet 干净测算待将来）。

### M3-4 · 回归与 CI eval
- eval 抽样接进 CI（architecture 已规划）；回归告警接线（`eval-criteria`「指标降>3pp 阻断」）。
- 交付：CI eval 抽样 + 回归门。依赖 M3-2 基线。

### M3-5 · 试用反馈闭环（dogfood）
- cron 常态运行一段、团队 dogfood、收集反馈迭代；多人交叉评审固化（承接 A1 条件④单人→多人）。
- 交付：运行记录 + 反馈清单 + 迭代。依赖 M3-1/2/3（稳定 + 达标）。

### M3-6 · 质量红线收紧（A1 遗留③）
- 幻觉率随规模收紧至 ≤2%（发布门）；analyzer 提示"数值须被本条引用覆盖"（#14 类）；截断守卫治本（streaming / 自由文本，A1 遗留）。
- 交付：幻觉率达标证据 + analyzer 改进。

### M3-7 · 收尾（低优）
- initial_digest 冷启动回填路径；deep_dive 的 LLM 叙述润色（架构留迭代）；视频字幕 / ASR spike。

## 外部依赖（不阻塞 M3 主体）
- **直连 `sk-ant-` key 目前不可得** → analyzer 切 Sonnet 降本、Anthropic 列表价干净成本测算留将来；M3 全程按 Opus-on-relay 现实推进。

## 不做（M3 out of scope）
- 多租户 / 完整权限、社媒源、实时时延、知识图谱沉淀（架构预留，M4+ 再议）。

## 建议起步顺序
**M3-1（稳定性地基）先行**——它解除 M3-2/3/5 反复撞到的超时阻塞，且完全不依赖 key。其后 M3-2（评测集）与 M3-3（成本）可并行。
