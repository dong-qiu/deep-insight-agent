## 愿景 (Vision)

让追踪一个快速演化领域的人，用每天几分钟读一份**可溯源、不拼凑**的洞察报告，
替代耗时且质量不稳定的手动综述 —— 趋势与最佳实践持续聚合、可复用。

## 探索性目标 (元目标)

除交付 Deep Insight 产品本身，本项目同样把「开发过程」作为目标 —— 用一个真实项目
作试验场，探索 AI 时代的软件开发新范式，并系统沉淀经验与教训。

探索主题（持续补充）：

- **人与 AI 协同的范式** —— 8 人团队 + Claude Code 如何分工、评审、决策
- **SPEC 的使用** —— spec 作为人类意图与 AI 实现之间的契约
- **Skill 的选择与使用** —— `skills/L0–L3` 分层约束的有效性
- **AI 生成代码的质量保障** —— 评审、测试、eval 如何应对 AI 代码
- ……（按实践推进补充）

> 自指特性：本项目的 `skills/`、IPD 流程、`specs/`、`decisions.md` 本身即实验对象；
> 而「AI 时代的软件工程」又恰是产品首批洞察行业之一 —— 项目在自己的领域里 dogfood。

**达成标志**：经验与教训被系统化沉淀于 `docs/practice-log.md`，且对后续项目可复用。

## 目标用户 (Target Users)

- **优先（当前）**：个人自用 —— dogfood 真实需求，反馈最快、不必妥协。
- **后续**：团队共用 —— 小圈子共享一份持续更新的洞察源。

典型画像：行业研究员 / 分析师、技术 / 产品从业者 —— 需要持续追踪某领域并产出可交付报告。
核心场景：持续追踪（每日 brief）+ 主题深挖。详见 `product-definition.md`。

## 关键问题 (Problem)

追踪一个快速演化的领域，今天的体验割裂且低效：

- **信息过载、来源分散** —— 资讯散落在网页 / 论文 / 社媒 / 视频 / 播客，无单一入口。
- **手动追踪成本高** —— 人工刷源再综述，耗时、易遗漏、难以持续。
- **现有工具各有短板** —— 聚合器只给信息流、不提炼；一次性深研不围绕主题持续演化；
  通用 LLM 总结幻觉风险高、不可溯源。
- **综述结果难复用** —— 手动综述是一次性产物，不沉淀、不累积、过时快。

## 关键假设 (Key Assumptions)

> 立项前提。每条需在 M1（计划）/ MVP 阶段验证；被证伪则需重新评估立项。

| # | 假设 | 验证方式 |
|---|---|---|
| A1 | LLM 能从多源噪音中可靠提炼出非显然、可溯源、低幻觉的洞察 | M1：真实数据小样本，人工核对引用一致性与洞察新颖性 |
| A2 | 目标用户愿意阅读并信任 AI 生成的报告（前提是可溯源） | MVP 后：团队 dogfood + 早期用户访谈 |
| A3 | 首批数据源在 API 成本 / ToS / 反爬上确实可达，且历史回填有足够可得深度 | M1：逐源做可行性核实（含历史回填深度） |
| A4 | "主题驱动持续聚合"相比"一次性深研"有显著增量价值 | MVP 后：对比 brief 与一次性报告的使用频次 |
| A5 | 单份报告的模型成本可控在可接受区间 | M1 估算 + MVP 实测 |

> **A1 验证状态（2026-05-26 · 有条件通过，证据加固、条件收窄）**：M2 早期真实数据实跑（详见 `docs/verify/a1-runs.md`）。
> A1 成立——LLM 能产出**可溯源、忠实、有用**的洞察，make-or-break 风险解除、准入 M2。
> 「非显然」经产品负责人定标由硬门槛降为追求层（见 `docs/verify/eval-criteria.md`）。
>
> **2026-05-26 复验（中转站已支持思考后）**：
> - **条件①闭合**：validator 带思考全量跑通（80.8%→82.1%，思考非瓶颈）；20/20 标注经人工校验正确（`docs/verify/a1-label-audit-2026-05-26.md`），故 82% 是对**可信标签**的真实准确率。负例召回 **100%** —— 校验器**安全（坏引用不放行）但保守（误杀好引用，损 yield 不损安全）**；90% 三分类门更属 yield 目标而非安全红线。
> - **条件④（提炼环）AI 预评 favorable**（`a1-ai-prereview-2026-05-26.md`，22 条）：非显然 ≥59%（含合格复述 100%）、**捏造率 ~0**，唯 #14 一条引用覆盖缺口（数值真实、应补引）。**待人工过工作表 `evals/out/review-sheet.md` 确认固化。**
>
> **剩余遗留条件（收窄）**：② 补非 arXiv 多源重测——边际价值已降（瓶颈是校验器保守、非源/标签），降级到 M3 发布门前或拿到直连 key 时做；③ 幻觉/引用覆盖随规模收紧至发布门 ≤ 2%（M3）；④ 人工过一遍洞察工作表确认数字。

## 不做什么 (Out of Scope)

战略边界（操作层完整清单见 `product-definition.md` 的「不做什么」）：

- ❌ 投资 / 买卖建议 —— 趋势预测仅作"基于证据的方向性判断 + 置信度"。
- ❌ 无引用的 AI 总结 —— 任何输出必须可溯源。
- ❌ 自动决策 / 自动行动 —— 系统只产出洞察，不触发外部动作。
- ❌ 内容创作 / 营销文案、辣评 / 风格化输出。
- ❌ 实时（秒级）时延 —— 报告以日 / 小时为粒度。
- ❌ 多租户 / 团队空间完整权限体系 —— 架构预留，当前不实现。

## 成功指标 (Success Metrics)

> 衡量产品价值，区别于 `docs/verify/eval-criteria.md` 的 AI 输出质量评分。
> 具体基线与目标值在 M1 / MVP 阶段标定。

- **北极星**：周有效阅读次数 —— 每周「有效阅读」的 brief / 报告数
  （有效 = 读完且至少发生一次 点击引用 / 追问 / 导出）。代表产品是否真正
  形成「持续追踪」的习惯与价值。
- **辅助指标**：
  - 深挖触发：人均每周主动发起的主题深挖 / 追问次数
  - 交付价值：报告被导出 / 分享 / 标记有用的比例
  - 效率：主题深挖出报告时延 P50 ≤ 10 分钟
  - 成本：单份报告平均成本 ≤ 预算阈值（M1 设定）
- **反向护栏指标**：错误引用率（特指引用—结论**一致性**失败率，非仅链接可达性）/ 幻觉率
  （人工抽检）必须低于阈值 —— 升高即视为退步。
