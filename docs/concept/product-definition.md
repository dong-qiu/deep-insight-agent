# 产品定义：Deep Insight

> 完整产品定义。状态：🟢 已基线 · Owner: dongqiu · 2026-05-17 · v0.7（2026-06-07：账号 MVP 口径收敛——MVP 为单用户，多账号数据隔离与自助注册推后，见「账号与权限」）

## 问题

追踪一个快速演化的领域（如 AI 软件工程、AI 安全），今天的体验割裂且低效：

- **信息过载、来源分散** —— 海量资讯散落在网页 / 论文 / 社媒 / 视频 / 播客，没有单一入口。
- **手动追踪成本高** —— 靠人工刷源再综述，耗时、易遗漏、难以持续。
- **现有工具各有短板** —— 聚合器（Feedly 类）只给信息流、不提炼；一次性深研（Perplexity / deep research 类）不围绕主题持续演化；通用 LLM 总结幻觉风险高、不可溯源。
- **综述结果难复用** —— 手动综述是一次性产物，不沉淀、不累积、过时快。

Deep Insight 要把"持续追踪一个领域"从耗时的手动劳动，变成可信赖、可溯源、可复用的自动洞察。

## 一句话

**Deep Insight 是一个面向行业的深度洞察系统**：自动采集业界资讯（公开网页 / 新闻、学术论文、社交媒体、视频、播客），提炼行业趋势与最佳实践，按主题输出**可溯源**的洞察报告。

**差异化（vs Perplexity / Feedly / 手动综述）**：
- 多源融合 —— 网页 / 论文 / 社媒 / 视频 / 播客同一引擎，不依赖单一搜索
- 可溯源 —— 每条结论可点回原文；引用经真实性与一致性双重校验，幻觉零容忍
- 主题驱动 —— 围绕"主题"持续聚合演化，而非一次性问答
- 趋势 + 实践并重 —— 既看动态，也提炼可复用的模式

## 核心能力

### 1. 采集 — 业界最新资讯

| 维度 | 设计 |
|---|---|
| 源类型 | 公开网页 / 新闻、学术论文（arXiv / 顶会 / 期刊）、社交媒体（X / 技术社区 / 订阅号 / 博客）、视频（YouTube / B 站 等）、播客（RSS / Apple Podcasts 等） |
| 接入方式 | RSS、官方 API、定向爬取、第三方搜索 / 学术 API、视频字幕 / 播客转写（ASR） |
| 触发 | 定时增量抓取（按源配置周期） + 按需触发 |
| 预处理 | URL 规范化、正文抽取、去重、标准化元数据（标题 / 作者 / 时间 / 来源 / 语言 / 标签） |
| 归档 | 原始内容 + 结构化内容双存，可回溯 |
| 容错 | 单源失败不阻塞流水线，记录错误并重试 |
| 配置化 | 源清单与接入方式通过配置维护，新增 / 调整 / 启停源无需改代码 |

### 2. 头脑风暴 — 洞察前的交互式探索

> 在正式洞察之前，与用户多轮交互，把模糊的兴趣方向收敛为可深挖的主题、关键问题与待验证假设。

| 维度 | 设计 |
|---|---|
| 触发 | 用户主动发起；也可由系统基于近期采集变化主动建议（"最近 X 方向有新动向，要不要展开？"） |
| 输入 | 用户初始想法 / 关键词 / 一段问题描述 |
| 交互形式 | 多轮对话为主；可选思维导图 / 卡片整理辅助收敛 |
| 数据上下文 | 可访问已采集的素材池作为背景，避免脱离实际信息空想 |
| 输出 | 结构化主题 + 关键问题清单 + 待验证假设 + 推荐数据切片，作为**洞察阶段的明确输入** |
| 留存 | 会话可保存 / 命名 / 续聊；可作为洞察报告的"前史"附在结果中，体现思考路径 |
| 模型可配 | 头脑风暴使用的 LLM 可独立配置（建议选对话强 / 创造性高的模型） |

### 3. 洞察 — 行业趋势与最佳实践

| 维度 | 设计 |
|---|---|
| 趋势识别 | 时间维度的热度变化、新兴主题发现、关联事件聚合 |
| 趋势预测 | 基于历史数据与多源证据，输出方向性判断（如"X 类技术正在加速 / 衰退"）；强制标注**置信度、依据、不确定性**；不做精确数字 / 具体时间点预测 |
| 最佳实践 | 从案例 / 论文 / 博客中归纳可复用模式（含适用场景与边界） |
| 主题聚合 | 跨源、跨语言整合同主题信息，自动归并近义说法 |
| 多维分析 | 横向对比（方案 / 玩家 / 路线）、纵向时间线、关键事件标注 |
| 实体追踪 | 组织 / 人物 / 项目 / 产品维度的动态聚合 |
| 信号去噪 | 重要性评分 + 阈值过滤；无重要事件时诚实说明，不凑数 |
| 模型可配 | 分析使用的 LLM 可配置（提供商 / 模型 / 参数），不同子任务可分别指定 |

### 4. 报告 — 按主题可溯源输出

| 维度 | 设计 |
|---|---|
| 触发方式 | 定时（每日 / 周 / 月）、按主题手动、按事件触发 |
| 内容结构 | TL;DR + 关键发现 + 趋势分析 + 对比表 + 时间线 + 引用清单 |
| 输出格式 | Markdown / HTML（自包含） / PPT（Marp） |
| 可溯源 | 每条事实和结论标注引用编号，可点回原文；引用做**双重校验** —— ① **可达性**（来源真实存在、可访问、被引片段确实在原文）② **一致性**（被引内容确实支持该结论，无断章取义 / 夸大 / 张冠李戴）；②为质量重点投入项 |
| 交互 / 交付 | 三种模式并存：① **单向输出**（直接阅读 / 下载 / 推送） ② **多轮追问**（基于已生成报告的上下文继续探索） ③ **沉淀为知识图谱**（实体 / 关系 / 主题随报告增长持续累积，跨报告复用） |
| 归档 | 报告按主题 + 日期归档，建立全局索引；**存储先用文件系统**（Markdown + JSON 索引），后续按规模 / 检索需求再评估升级 |
| 搜索 / 筛选 / 排序 | 所有报告（含每日 brief 与主题深挖）支持全文搜索（标题 + 正文 + 摘要）、多维筛选（主题 / 日期 / 来源 / 标签 / 实体 / 报告类型）、多维排序（时间倒序默认 / 时间正序 / 重要性 / 标题 / 主题） |
| 推送 | 邮件 / IM（Slack / Telegram / 微信 Bot 等）；渠道启停、目标地址、推送规则均可配置 |
| 不复报 | 每日 brief 不重复报告历史 brief 中已出现的事件；同事件的**后续进展**可作为"更新"再次出现，并标注前情链接 |

### 5. 蓝军分析 — 对给定报告 / 文章的批判性审视

> 按需触发的补充能力。对系统内已生成的报告或外部给定的文章发起"反方视角"分析。

| 维度 | 设计 |
|---|---|
| 输入 | 系统内已生成的报告、外部 URL、用户上传的文件 / 文本 |
| 分析视角 | 薄弱论点、缺失证据、隐含假设、反例 / 反证、替代解释、潜在风险与盲区 |
| 输出 | "蓝军视角"评估报告：逐条指出问题 + 反方论据 + 改进建议 + 引用支撑 |
| 中性约束 | 基于证据与逻辑的结构化批判，**不做情绪化辣评**；仍遵守可溯源、不幻觉 |
| 多轮 | 支持就蓝军报告再追问；可形成"原文 ↔ 蓝军 ↔ 红方反驳"多轮对照 |
| 关联归档 | 蓝军报告与原报告 / 文章双向关联，可并列对比阅读 |
| 模型可配 | 蓝军分析使用的 LLM 可独立配置（可选用更强推理模型） |

## MVP 范围

> 首版（MVP）的目标是「让系统端到端跑起来」—— 验证核心链路成立，不是功能完整。
> 对应里程碑见 `docs/plan/roadmap.md`。

### MVP 包含

| 能力 | MVP 范围 |
|---|---|
| 采集 | 仅接入确定可达的源（RSS / arXiv / 官方 API）；定时增量抓取 + 基础预处理 + 接入时一次性历史回填（见「冷启动」） |
| 洞察 | 主题聚合 + 趋势识别 + 信号去噪 + 多源交叉；先跑通单行业、少量主题 |
| 报告 | 每日 brief + 主题深挖报告；可溯源引用 + 引用校验；Markdown / HTML 输出；基础全文搜索 |
| 账号 | 最小登录：单用户单管理员登录（env 凭据）。**自助注册与多账号数据隔离推后**——MVP 为单用户，数据模型暂不引入 userId 维度（见「账号与权限」与「MVP 暂不包含」） |
| UI | 报告阅读页 + 最小 Brief + 最小设置 |

### MVP 暂不包含（后续迭代）

| 能力 | 说明 |
|---|---|
| 头脑风暴 | 洞察的可选前置，非首版必需 |
| 蓝军分析 | 补充能力，后续接入 |
| 知识图谱 | 需数据累积，后续接入 |
| 趋势预测（带置信度的方向性判断） | 趋势识别先做，预测能力后做 |
| 推送（邮件 / IM） | 首版后优先项 |
| 多源扩展（社媒 / 视频 / 播客） | 按数据源可行性逐步接入 |
| 管理看板完整形态 | 首版仅保留最小可观测，完整看板后续做 |
| 自助注册 / 多账号数据隔离 | MVP 为单用户（单管理员登录）；自助注册与按账号隔离（reports/topics/sources 引入 userId 维度）一并推后。数据模型预留演进空间但暂不落地 |
| PPT 导出、多租户 / 团队空间 | 更后 |

> 本节是 MVP 的范围约定；逐版本的范围细化在各 spec 与 `roadmap.md` 中维护。

## 目标用户

- **优先（当前）**：**个人自用** —— dogfood 自己的真实需求，反馈最快、不必妥协
- **后续**：**团队共用** —— 小圈子共享一份高质量、持续更新的洞察源

典型画像（自用与团队场景共通）：
- **行业研究员 / 分析师** —— 需要持续追踪某领域并输出可交付报告
- **技术 / 产品从业者** —— 关注特定方向的趋势、实践与竞品动态

> 架构上需为多用户预留扩展空间（账号 / 隔离 / 权限），但当前实现以单人体验为优先，不为未来过度设计。

## 行业范围

**当前聚焦**（首批落地的两个领域）：
- **AI 时代的软件工程** —— AI coding、Code Agent、IDE 集成、Skill / MCP 等基建、研发范式演进
- **AI 时代的安全** —— AI 系统攻防、模型安全、Agent 安全、AI 辅助攻防、对抗与防御

**后续扩展**：架构层面支持新增行业领域；新行业可通过新增"主题 + 数据源 + Prompt 模板"配置接入，无需改核心代码。

## 首批数据源（按行业推荐）

> 推荐目标源清单。MVP 先接入其中确定可达的子集（RSS / arXiv / 官方 API），其余按可行性逐步接入。所有源均可在配置中增删 / 启停 / 调整频率。

### AI 时代的软件工程

- **综合社区**：Hacker News（高质量技术讨论）
- **学术**：arXiv（cs.SE / cs.CL / cs.AI）
- **厂商博客**：Anthropic、OpenAI、Google Research、Cursor、GitHub Next
- **Newsletter / 播客**：Latent Space、The Pragmatic Engineer、Practical AI
- **社交媒体（X）**：@swyx、@simonw、@karpathy
- **视频**：Andrej Karpathy YouTube、3Blue1Brown
- **中文**：机器之心、量子位、InfoQ AI 频道

### AI 时代的安全

- **综合资讯**：The Hacker News、KrebsOnSecurity、Bleeping Computer
- **学术**：arXiv（cs.CR）、USENIX Security、IEEE S&P 论文集
- **标准 / 框架**：OWASP LLM Top 10、OWASP AI Exchange、MITRE ATLAS（AI 威胁矩阵）
- **厂商博客**：Anthropic Safety、OpenAI Safety、Google Security blog
- **视频 / 播客**：DEFCON / BlackHat YouTube、Risky Business、Darknet Diaries

## 用户场景

- **持续追踪**：每日 / 周自动扫描，按**配置的主题**输出 brief（主题清单可增删、启停、调整频率），5 分钟掌握关键事
- **头脑风暴**：在正式洞察前与系统多轮对话，把模糊兴趣收敛为可深挖的主题、关键问题与待验证假设，再交给洞察引擎深入处理
- **主题深挖**：输入任意主题 → 5–10 分钟拿到结构化报告，可发给同事
- **事件跟进**：重大事件发生后，系统持续聚合后续进展，形成事件专题
- **历史回溯**：按主题 / 时间 / 实体检索历史洞察，复用过往结论
- **多轮追问**：在已生成报告基础上提问，系统基于上下文补充分析

## 冷启动与数据积累

> 部分能力依赖数据随时间累积，系统在最初几天 / 几周明显弱于稳态。此处显式设计，不回避。

| 能力 | 冷启动影响 | 应对 |
|---|---|---|
| 趋势识别 / 预测 | 无时间跨度即无"变化"与"历史" | 新主题 / 新源接入时做一次性历史回填（范围按源可得性与成本设定） |
| 知识图谱 / 实体追踪 | 初期为空，需累积才有价值 | 能力渐进可用，UX 标注"积累中"，随数据量解锁 |
| 每日 brief / 不复报 | 首次扫描面对全部历史，易一次性信息过载 | 首份 brief 定义为"初始摘要"而非日常 brief，并以此设定"不复报"基线 |

> 原则：未达数据量的能力对用户**诚实标注"积累中"**，不伪装成稳态结果（呼应「质量原则 · 诚实兜底」）。

### 设计原则

> 由「质量原则」延伸到界面层。

| 原则 | 含义 |
|---|---|
| 可溯源优先 | 每条结论的引用"一键可达"：行内标记 → 悬浮预览 → 来源面板回原文 |
| 阅读优先 | 核心交付物是一份供人读的报告；排版、可扫读性、TL;DR 上浮第一优先 |
| 诚实的空状态 | "无重要事件"明确呈现，不用占位内容填空 |
| 进度透明 | 采集 / 分析 / 生成是分钟级异步任务；展示流水线进度，不给空转圈 |
| 渐进披露 | 默认简单，深度按需展开（多轮追问、对比、知识图谱） |
| 克制中性 | 研究工具气质，内容优先；不做营销风花哨动效 |

### 信息架构（主导航）

| 区域 | 作用 |
|---|---|
| 今日 Brief | 落地页：每日概览，5 分钟掌握关键事 |
| 主题 | 已订阅主题，每个主题持续聚合演化 |
| 报告库 | 全部报告，搜索 / 筛选 / 排序 |
| 头脑风暴 | 多轮对话工作区，收敛主题 / 问题 / 假设 |
| 蓝军 | 批判性审视，原文 ↔ 蓝军对照 |
| 知识图谱 | 实体 / 关系 / 主题的累积视图 |
| 看板 | 管理员：报告生命周期、流水线、成本（普通用户只看自己的） |
| 设置 | 数据源、主题、推送渠道、模型偏好、成本配额 |

### 核心界面

| 界面 | 设计要点 |
|---|---|
| 报告阅读页 | 产品心脏。TL;DR + 关键发现 + 趋势 + 对比表 + 时间线 + 引用清单；行内引用 `[1]` 可点；侧边目录；页内"继续追问"入口 |
| 今日 Brief | 卡片化关键事，每条带主题 / 重要性 / 来源数；不复报已读事件，后续进展标"更新" |
| 报告库 | 全文搜索 + 多维筛选（主题 / 日期 / 来源 / 标签 / 实体 / 类型）+ 多维排序；列表 / 卡片双视图 |
| 头脑风暴工作区 | 多轮对话为主，可选思维导图 / 卡片辅助；侧栏实时收敛"主题 + 关键问题 + 假设" |
| 主题页 | 该主题的报告时间线、关键实体、重大事件标注 |
| 蓝军对照页 | 原文 ↔ 蓝军视角并列，支持"红方反驳"多轮对照 |
| 管理看板 | 报告状态流、端到端流水线追踪、数据源健康、成本时序图；失败任务可下钻 + 重试 |
| 设置 | 配置密集：数据源增删启停、主题管理、推送渠道、按子任务的模型选择 |

### 关键交互模式

| 模式 | 设计 |
|---|---|
| 引用交互 | `[n]` 悬浮 = 预览卡，点击 = 打开来源面板 / 原文；来源类型用图标区分（网页 / 论文 / 社媒 / 视频 / 播客） |
| 多轮追问 | 任意报告页内发起上下文追问，不跳出阅读场景 |
| 异步任务反馈 | 流水线步进器（采集 → 分析 → 报告）；任务可后台运行，完成后通知 |
| 流式输出 | 报告 / 分析结果流式呈现，降低等待焦虑 |
| 推送落地 | 邮件 / IM 推送点进 = 直达对应报告的 deep-link，移动端可读 |
| 状态语言 | 加载 / 空 / 错误三态显式且诚实，错误可一键重试 |

### 多端策略

| 端 | 定位 |
|---|---|

## 账号与权限

- **需要账号体系**：用户登录 / 身份识别为系统基础能力（**目标态**含自助注册）
- 个人配置（订阅主题、数据源偏好、推送渠道、模型偏好）按账号隔离（**目标态**）
- 报告与历史记录按账号隔离访问；公共主题报告可共享（**目标态**）
- 多租户 / 团队空间 / 角色权限（admin / member / readonly）：当前不做，架构预留（见「不做什么」）
- **MVP 口径（2026-06-07 收敛）**：MVP 为**单用户**——单管理员登录（env 凭据，见 `src/auth.ts`），数据模型暂不引入 userId 维度；自助注册与多账号数据隔离一并推后。上述三条"按账号隔离/注册"为**目标态**，待引入多用户时落地。理由：当前以个人自用 dogfood 为优先（见「目标用户」line 134），不为未来过度设计；架构层保留演进空间但不预先实现。

- 分析与报告输出支持中文 / 英文 / 中英混合（按用户配置或主题语言自适应）

| 维度 | 设计 |

| 数据源健康 | 各源的成功率、最近抓取时间、内容量级、近期错误模式；自动告警（连续失败 / 限流） |

## 质量原则

| 原则 | 含义 |
|---|---|
| 可溯源 | 任何结论必须可点回原文，不做无引用的"AI 总结"；引用须过可达性 + 一致性双重校验 —— "可点回原文"不等于"原文支持该结论" |
| 诚实兜底 | 信息不足时直说"无重要事件"或"信息不充分"，不凑数 |
| 多源交叉 | 重要结论需多源印证；单源信息明确标注 |
| 中性叙述 | 客观呈现事实，不主观臆断、不编辑评论、不带情绪；趋势预测仅以"基于证据的方向性判断 + 置信度"形式呈现（见「核心能力 · 洞察」） |
| 可解释 | 重要性评分、聚类决策可追溯到规则或证据 |

## 安全设计

> 安全在概念阶段统一纳入，不作为后期补丁；面向 AI agent 类系统的特定威胁（prompt injection、模型滥用、数据泄露）有专门约束。

| 维度 | 设计要求 |
|---|---|
| 身份与隔离 | 账号鉴权 + 会话管理；管理员后台独立鉴权。配置 / 数据 / 报告按账号严格隔离为**目标态**，**MVP 推后**（MVP 单用户，不引入 userId 维度，见「账号与权限 · MVP 口径」） |
| 密钥管理 | 模型 API key、第三方 token、推送凭据集中加密存储；不入日志、不在前端暴露；最小权限原则 |
| 传输与存储 | 全链路 TLS；敏感字段加密存储；备份与灾备可恢复。删除账号时彻底清除关联数据为**目标态**，**MVP 推后**（MVP 单用户，无多账号注销场景，见「账号与权限 · MVP 口径」） |
| 输入防护 | **抓取内容做 prompt injection 防护**（隔离上下文 / 标注来源 / 剥离指令式文本）；用户输入防 XSS / 注入 |
| 输出防护 | 不泄露其他用户数据 / 系统配置 / 内部 prompt；引用 URL 做安全检查（恶意链接 / 钓鱼）；模型输出过滤敏感信息 |
| 合规与版权 | 抓取遵守 robots.txt / 各源 ToS；引用必标来源。账号注销与数据删除（满足主流隐私法规要求）为**目标态**，**MVP 推后**（随多账号能力一并落地，见「账号与权限 · MVP 口径」） |
| 供应链 | 依赖锁定 + 漏洞扫描；构建可重现；模型来源可信；第三方组件定期更新 |
| 审计与日志 | 关键操作（登录、配置变更、源接入、报告生成、推送、删除）全部记录可追溯日志；敏感字段脱敏 |
| 防滥用 | 按账号 / IP / 主题做速率限制（与「成本与配额」联动）；异常行为告警 |

## 不做什么

> 明确边界，避免范围蔓延。

- ❌ **投资建议 / 买卖建议** —— 不给出任何金融操作建议（趋势预测以"基于证据的方向性判断 + 置信度"形式呈现，不构成投资建议）
- ❌ **无引用的 AI 总结** —— 任何输出必须可追溯
- ❌ **自动决策 / 自动行动** —— 系统只产出洞察，不触发外部动作
- ❌ **内容创作 / 文案生成** —— 输出是分析报告，不是营销 / 公关文案
- ❌ **辣评 / 风格化输出** —— 保持中性
- ❌ **实时交易级时延** —— 报告以日 / 小时为粒度，不追求秒级
- ❌ **多租户 / 团队空间完整权限体系** —— 架构预留，当前不实现
