# A1 验证实跑记录

> charter 关键假设 **A1**（「LLM 能否可靠提炼非显然/可溯源/低幻觉的洞察」，DCP-1→M2 硬门槛）的实跑留痕。
> 每次跑追加一条。评测口径见 `eval-criteria.md`，harness 见 `/evals`。

## A1 结论（2026-05-25 · 有条件通过）

经三轮真实数据实跑 + AI 预评 + 负责人初判，A1 判定**有条件通过**，准入 M2。

- **成立的核心**：LLM 能产出**可溯源（引用可达性 100%）、忠实准确、有用**的行业洞察，make-or-break 风险解除。
- **校准**：「非显然」由硬门槛（≥60%）降为**追求层**（目标 ≥30%，本轮 ~45% 已超；单篇忠实复述亦属合格）——产品负责人定标，可达性/幻觉红线不松（见 `eval-criteria.md` 2026-05-25 校准）。
- **遗留条件（M2 内闭合）**：
  1. validator 一致性始终未真正验证（中转站跑不了带思考）→ 用直连 `sk-ant` key 带思考补验；
  2. 评测数据全 arXiv 单源 → 补 news/社交/视频等多源重测，确认结论稳健；
  3. 幻觉/引用覆盖：当前小样本约 5%（1 例 SMDD 40.2% 引用未覆盖；另 2 例截断已被产出守卫拦截）→ 随规则 4 + 守卫持续收紧，发布门（M3）前压到 ≤ 2%；
  4. 人评为单人快读印象 → 建议补一轮多人正式打分（CSV `a1-review-2026-05-25.csv` 已就绪）固化数字。

## 2026-05-24 · 首次真实数据全量跑（经第三方中转站）

**配置**：analyzer=`claude-opus-4-6` / validator=`claude-opus-4-7`，**validator 无思考**（`VALIDATOR_THINKING=0`——中转站对「非流式 + adaptive thinking 的长响应」会卡死）。
数据：5 主题 × 真实 arXiv 摘要（30 条）、120 组标注一致性对。接入：第三方中转站 `yibuapi.com`（非 Anthropic 直连）。

**自动指标**：

| 指标 | 实测 | 门槛 | 结果 |
|---|---|---|---|
| 引用可达性通过率 | 100% | =100% | ✅ |
| 引用一致性合格率 | 58.1% | ≥95% | ❌ |
| 一致性失败率 | 32.3% | ≤5% | ❌ |
| flagged 率 | 9.7% | ≤10% | ✅ |
| 校验器三分类准确率 | 80.8% | ≥90% | ❌ |
| 校验器负例召回率 | 98.3% | ≥95% | ✅ |

**成本**：$1.27（约 155 次 opus 调用；Anthropic 列表价估算，中转站实扣以其计价为准）· $0.04/引用校验。

**结论：管线 + 成本验证通过，A1 尚未判定。** 失败的三项高度受污染，**不可作 A1 结论**：
1. validator 被迫无思考（中转站限制）→ 三分类准确率仅 80.8%、偏向「宁可误杀」（负例召回 98.3%）→ 很可能把忠实引用误判成 not_support，抬高了 58%/32%；
2. 一致性标注由 AI 构造、未经人工校验（衡量的是 LLM-vs-LLM-标签）；
3. 复合句洞察挂多条引用，逐条判「该引用是否支持整句」时易判 not_support。

**可信信号**：引用可达性 **100%**（可溯源成立）；抽查洞察能跨多篇综合、引用逐字忠实，质量观感良好（待系统人评确认）。

**已知瑕疵**：
- Code Agent 主题因 analyzer 调用超时被整个跳过 → 人评队列为 18 条洞察 / 4 主题（非 23）。
- `multi_source` 在全 arXiv 数据上恒为 false（`source_count` 数的是源类型，全 arxiv→1），该标志在单源数据上无意义。

**待办（拿到可信 A1 结论前）**：
1. 人评 18 条洞察的**非显然占比 / 幻觉率**（工作表 `evals/out/review-sheet.md`，`npm run review:sheet` 生成）；
2. 换**直连 `sk-ant-` key** 让 validator 带思考后补跑（顺带可用回 Sonnet 分析档、数据不出境）；
3. 一致性标注集补**人工校验**（非 AI 构造标签）；
4. 补跑被跳过的 Code Agent 主题；
5. 评测集补**非 arXiv 源**（news/社交/视频），才能真正压测多源去噪。

## 2026-05-24（二）· 改进 analyzer 提示后重跑（同中转站）

**改动**：analyzer SYSTEM 加规则 4–10（引用覆盖结论 / 不得放大 / 完整自足 / 偏好非显然综合 / 去重 / trend 约束）。配置同上轮（opus-4-6 分析 / opus-4-7 校验、validator 无思考）。成本 $1.34。

**自动指标（vs 上一轮）**：

| 指标 | 上轮 | 本轮 |
|---|---|---|
| 自动门槛 | 3/6 | 1/6 |
| 引用可达性 | 100% | 97.3% |
| 一致性合格率 | 58.1% | 45.9% |
| 一致性失败率 | 32.3% | 21.6% |
| flagged 率 | 9.7% | 29.7% |
| 校验器三分类准确率 | 80.8% | 81.2% |
| **跨源综合洞察占比** | ~22%（4/18） | **~56%（9/16）** |

**关键判断：auto-metrics 表面变差，主因是「综合度提升」撞上「无思考 validator」。** 洞察更复合 → 无思考校验器更频繁判 uncertain（flagged 9.7%→29.7%）；而校验器对**固定标注集**的准确率几乎没变（80.8%→81.2%），证明不是校验器变差，是它在没思考时判不动更复合的结论。**提示改进在最要紧的维度（跨源综合/非显然）明显见效**——出现了跨领域综合（如「地理空间+医疗 QA 均在多源推理上退化」「RAG 上游 Web 提取 + 底层向量检索双瓶颈」「coding agent 的 reward hacking + 越权两类失控面」）。

**AI 复评（同源偏好，仅 triage）**：
- ✅ 综合度大涨（~22%→~56%）；TIDE「无损」声明这轮已被引用覆盖（规则 4 见效）。
- ❌ **截断未修**：医疗 RAG 那条仍截断成「…面向高风险医疗问答的」——疑似模型对「高风险医疗」敏感词中途停笔（安全行为，非 max_tokens）。
- ❌ **Prompt Injection 主题整个被安全拒答**（stop_reason=refusal），丢 5 条。
- ⚠️ 1 条引用可达性失败：WCXB 那条把 body 里**不连续**的两句拼成一条 quote。
- ⚠️ SMDD「40.2%」声明仍未被所挂 quote 覆盖（规则 4 只部分见效）。

**据此再修（本次提交）**：
1. analyzer 系统提示加**敏感领域框定**（安全/医疗等是中性分析对象，不得拒答/中途停笔）——同时针对拒答与截断（疑均为安全行为驱动）。
2. 规则 3 加 **quote 必须连续**（治拼接导致的可达性失败）。
3. LLM client 对 `stop_reason=refusal` **重试至多 3 次**（拒答多为非确定性）。

效果待下一轮验证。

## 2026-05-25 · 验证三处修复（run-3，同中转站）

验证 run-2 后提交的三处修复。配置同前。成本 $1.56。**3 处修复 2 成 1 败。**

- ✅ **Prompt Injection 拒答已修**：该主题正常产出 5 条（run-2 整个被 refusal）。框定 + refusal 重试见效；**5 主题全产出、共 22 条洞察**（三轮最全）。
- ✅ **可达性回到 100%**：0 条引用未命中（run-2 的 WCXB 拼接被「quote 必须连续」规则修复）。
- ❌ **截断未修，且假设被证伪**：本轮 2 条截断——医疗 RAG 那条仍截断，**外加一条出现在 Prompt Injection 的纯技术句**（「…嵌入最小间距仅为 1.02，存在显著的」就断）。后者**不是敏感领域** → 截断**不是安全行为驱动**（上轮假设错误），更像**结构化输出把长 statement 字符串提前收尾**（随后照常补全 JSON 其余字段，故 JSON 仍合法）。规则 6 + 敏感域框定均无效。

**指标（三轮对比）**：

| | run1 | run2 | run3 |
|---|---|---|---|
| 门槛 | 3/6 | 1/6 | 2/6 |
| 可达性 | 100% | 97.3% | 100% |
| 跨源综合 | 22% | 56% | 45%（10/22）|
| validator 三分类准确率 | 80.8% | 81.2% | 79.5% |

（综合占比从 56% 回到 45% 是因 PI 主题回归、带回几条单篇洞察摊薄分母，非能力倒退；validator 仍无思考、不可信照旧。）

**截断 = 当前首要 bug，且非 prompt 可解**（结构化输出层面的字符串过早终止）。处置：本次按方案(1)加**产出守卫**——analyzer 产出后检测半句 statement（结尾非句末标点）即**丢弃并告警**，不让其污染指标与人评。治本方向（自由文本输出+自行解析 / 长输出走 streaming）留后续。
