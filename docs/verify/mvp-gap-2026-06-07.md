## 一、硬缺口：MVP 范围里写了、但没做或只做一半

### 1.1 账号体系 —— 只有单管理员，注册与数据隔离缺失 → ✅ 已决策推后（2026-06-07）

- **承诺（原）**：`product-definition.md` 旧版「最小账号体系（注册 / 登录 / 数据隔离）」；安全设计 line 324「配置 / 数据 / 报告按账号严格隔离」。
- **实际**（`src/auth.ts:15-26`）：单管理员，凭据来自环境变量 `ADMIN_EMAIL/ADMIN_PASSWORD`，JWT session；**无注册流程**。
- **根因层级**：reports / topics / sources 等表**没有 userId 维度**——`src/app/page.tsx`、`src/app/settings/page.tsx` 直接取全量数据。"按账号隔离"在数据模型层就不存在，不是 UI 未接。
- **决策（2026-06-07）**：**MVP 口径收敛为单用户**——单管理员登录即满足，自助注册与多账号数据隔离推后。已落到 `product-definition.md` 六处（v0.7）：①MVP 包含 账号行重写；②MVP 暂不包含 新增"自助注册 / 多账号数据隔离"行；③账号与权限节"按账号隔离"标为目标态 + 加 MVP 口径说明；④安全设计「身份与隔离」行隔离要求标为目标态/MVP 推后（保留鉴权/会话/管理员独立鉴权在范围内）；⑤安全设计「传输与存储」行"删除账号时清除关联数据"标为目标态/MVP 推后；⑥安全设计「合规与版权」行"账号注销与数据删除"标为目标态/MVP 推后。**口径不一致已闭合**——不再是缺口，而是显式推后项。

### 1.2 成本配额 / 上限告警 / 限流 —— 有度量，无管控 🟡

- **承诺**：`product-definition.md:288-291`「可配成本上限（月/日/单次）；触顶告警 + 限流（暂停非关键任务）；成本可视化」。
- **实际**：度量侧已做（`src/lib/runtime/run-stats.ts` / `cost.ts` / `ops/cost-backfill.mjs`）；**可配置上限 + 触顶自动限流/暂停采集**未落地。`src/lib/runtime/rate-limit.ts` 是请求级限流，非成本配额。
- **影响**：Opus-on-relay 含校验单轮 ¥14–26（`roadmap.md:92`）的成本结构下，**无刹车**，dogfood 期间靠人盯。

### 1.3 真实成本账单未定稿（外部依赖阻塞）🟡

- roadmap M4 附条件①：直连 `sk-ant-` key 不可得 → Sonnet 降本测不了、Anthropic 列表价干净测算做不了，validator 锁在 Opus relay 上是成本大头。
- **判定**：已知、被外部依赖阻塞，非遗漏。

## 二、软缺口：MVP 划为"暂不包含"，但 dogfood 已叩门

> `product-definition.md:110-121` 列为"后续迭代"，严格非 MVP 欠缺，但 daily dogfood 体验立刻撞上。

### 2.1 推送（邮件 / IM）—— 连"做完通知我"都没有 🟡

- 失败告警只接 `webhook.site` 占位 + `ops/probe-alert.mjs` 探针（`roadmap.md` M4 ②）；**真渠道 adapter（ntfy/飞书/Bark/PushDeer）待开发**；**报告推送**（brief 推给用户）整体缺席。
- dogfood 核心关注是"你会不会主动来读 brief"（`dogfood-log.md:37`）。无推送 = 每天得自己记得开页面，对"5 分钟掌握关键事"的主张是结构性削弱。

### 2.2 "主题"导航与主题页缺失 🟡

- IA（`product-definition.md:217`）把「主题」列为主导航，主题页应有"报告时间线 / 关键实体 / 重大事件标注"。
- 实际导航仅 4 项（`src/app/layout.tsx:17-20`），**无主题页**；主题只能在 settings 里 CRUD。"围绕主题持续聚合演化"是护城河之一（DCP-1 备忘 `roadmap.md:74`），UI 上无承载面。

## 三、体验缺口：链路通了，手感未到位（dogfood 已记录）

### 3.1 引用交互停在"跳列表"，无 hover 预览 🟢

`product-definition.md:243` 设计「`[n]` 悬浮=预览卡，点击=来源面板」；当前点 `[N]` 跳底部引用列表（A1 观察焦点 `dogfood-log.md:23`）。已把 `[N]` 渲染成可见蓝色序号 + quote 链接（`dogfood-log.md:53-54`），hover 预览未做。

### 3.2 多轮追问（A4）入口缺席 🟢

报告页设计「页内继续追问」（line 230），实际报告页无此入口。这是 dogfood 当前验证"是否真需要"的头号假设（`dogfood-log.md:37`）——先观察再决定，合理。

### 3.3 进度透明 / 异步任务反馈 🟢

设计要求「流水线步进器 采集→分析→报告，不给空转圈」（line 245）。深挖/立即抓取后目前要去 /admin 数 Run 表（`dogfood-log.md:24`），无实时进度。
